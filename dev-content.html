Welcome

Havok is open source, so it is yours to enjoy, change and improve. We love every contribution, from grammar corrections in the documentation to the most hardcore code.

Havok is hosted on github. You can post issues, suggestions and contribute to discussions there.

Setup

To develop havok you will need havok will all development dependencies installed. To do this, clone havok and install with npm:

  1. git clone http://github.com/zoopcommerce/havok
  2. cd havok
  3. npm install

dev server

The development server is found in havok/dev. It is a http server run on node.js set up for easy testing and development of havok. You'll find more instructions below.

Pull Requests

To add your fixes or improvements to Havok you'll need to open a pull request in github.

To open a pull request, first fork havok on github.

Then do the following to save your local changes up to your github fork:

  1. #create a new branch to hold your changes
  2. git checkout -b my-patch
  3.  
  4. #add your fork as a new remote
  5. git remote add fork http://github.com/username/havok
  6.  
  7. #stage all your changes for commit
  8. git add .
  9.  
  10. #commit changes
  11. git commit -m "Description of changes made"
  12.  
  13. #push your changes to github
  14. git push fork my-patch

Finally, go to your github fork, and click 'open pull request'

Thanks for your contribution!

Runing Tests

Havok uses the intern test runner.

node.js

To run tests in node, use:

  1. node path/to/havok/test/runtests.js

Browser

To run tests in a browser, first start the havok dev server:

  1. node path/to/havok/dev/server.js runtests

Then point your browser to:

  1. http://localhost

Note: when running in a browser, the output will be in the browser console.

Docs

Havok documentation is generated from the twig templates found in docs/twig.

Browsing docs

To browse the unbuilt havok docs, first start the havok dev server:

  1. node path/to/havok/dev/server.js docs

Then point your browser to:

  1. http://localhost

Building docs

To build the documentation, run:

  1. node path/to/havok/docs/make.js

This will a build havok layer for the documentation, and save all twig templates as flat html files.

The built docs will be found in havok/docs/dist.

Browsing built docs

To browse the built havok docs, first start the havok dev server:

  1. node path/to/havok/dev/server.js dist-docs

Then point your browser to:

  1. http://localhost

Other docs tools

The following scripts are used by the docs build tools, but may be run separately if you wish:

  • node path/to/havok/dev/docs/apiParse.js Parses havok js and creates tree.json and details.json.
  • node path/to/havok/dev/docs/apiStoreWiter.js Parses tree.json and creates the data store used by the api docs tree navigation widget.
  • node path/to/havok/dev/docs/apiDetailsWiter.js Parses details.json and creates individual json files for each api documentation page.
  • node path/to/havok/dev/docs/apiGenerateDocs.js Runs the apiParser, apiStoreWriter, and apiDetailsWriter in turn.
  • node path/to/havok/dev/docs/buildLayer.js Builds the docs.js layer.
  • node path/to/havok/dev/docs/generateHTML.js Converts the twig templates into flat HTML files.

Builds

To build havok run:

  1. node path/to/havok/build/make.js

The result will be found in

  • havok/release contains havok and all dependency packages as individual files after they have been passed through the build tools.
  • havok/dist contains a compressed havok layer containing the dojo loader, all havok modules, and all dependencies. Also contains compressed css for the havok layer.
