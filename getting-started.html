Download Havok

Get havok files

Quick and Dirty

If you just want to take havok for a quick spin, the simplest way is to download the two havok distribution files. These contain all of havok, and much of dojo. They are big and heavy, but it's a simple way to try things out:

  • havok.js
  • havok.css

Full sources with npm (recommended)

npm install havok

Full sources with composer

First get composer with:

curl -sS https://getcomposer.org/installer | php

Then add the following composer.json file to the root of your project:

{
    "name": "my/project",
    "repositories": [{ "type": "composer", "url": "http://zoopcommerce.github.io/pixie"}],
    "require": {"havok/havok" : "~1.0"},
    "extra": {"zoop-js-path": "public/js"}
}

Then run composer in your project root:

php composer.phar update

Havok and dependencies will be insalled into myproject/vendor and symlinked to myproject/public/js

File structure

If you have downloaded the source you'll find the following file structure and contents:

havok/
├── .travis/   scripts used by travis.ci
├── build/     havok build tools
├── dev/       tools for developing havok itself
├── dist/      compressed build which includes every havok component and supporting css
├── docs/      unbuilt documentation files
├── lib/       all havok modules live here
└── test/      havok unit tests

Loading

Load Havok in your pages.

Quick and Dirty

If you are using the Havok dist you need to link havok.css, and load havok.js:

  1. <link rel="stylesheet" href="path/to/havok.css">
  2. <script src="path/to/havok.js"></script>

Simple development confiuration

Configure havok so you can develop your own javascript modules against it. In development mode all js modules are loaded async. In addition, less files are also loaded async and compliled client side. This makes development mode very flexible, responsive to code changes, and simple to debug. However, it does mean page loads are slow, so don't use this for production code.

First build havok:

  1. node path/to/havok/build/make.js

When the build is complete, you will find havok and all havok dependencies nicely arranged in the path/to/havok/release directory.

Add the following to your page, and you'll be ready to rock:

  1. <script type="text/javascript">
  2. 	dojoConfig = {
  3. 		isDebug: true,
  4. 		async: true,
  5. 		deps: ['havok/ready!'],
  6.         merge: ['havok/config']
  7. 	}
  8. </script>
  9. <script src="havok/release/dojo/dojo.js"></script>

To add your own AMD package to the dojo loader, use:

  1. <script type="text/javascript">
  2. 	dojoConfig = {
  3. 		isDebug: true,
  4. 		async: true,
  5. 		deps: ['havok/ready!'],
  6.         merge: ['havok/config'],
  7.         packages: [
  8.             {name: 'mypackage', location: 'path/to/my/package'}
  9.         ]
  10. 	}
  11. </script>
  12. <script src="havok/release/dojo/dojo.js"></script>

Including deps: ['havok/ready!'] will bootstrap havok. For more information about bootstrapping read about havok/ready!

Including merge: ['havok/config'] will use the default havok configuration. For more information read about the config merger

Advanced development configuration

You can load havok, dojo, and all dependencies without first building havok. This is particularly useful if you also intend to work on havok source code itself.

To do so requires remapping paths on the server side.

Add the following to your page:

  1. <script type="text/javascript">
  2. 	dojoConfig = {
  3. 		isDebug: true,
  4. 		async: true,
  5. 		deps: ['havok/ready!'],
  6.         merge: ['havok/config']
  7. 	}
  8. </script>
  9. <script src="dojo/dojo.js"></script>

You will then need to map requests for AMD modules to their correct file locations. If you are using node.js, then the havok/dev/packages.js can help you do this.

Production configuration

It is strongly recommended that you use the havok build tools to create a bootable layer that is used in production. Load your built layer css and js, and you're ready to go:

  1. <link rel="stylesheet" href="path/to/layer.css">
  2. <script src="path/to/bootable/layer.js"></script>

Using Havok

Once havok is loaded, start using it!

Declarative

The simplest way to use havok is by adding tags that will be rendered as widgets by the havok parser. Eg:

  1. <w-checkbox-button-group>
  2.     <button class="btn-primary">Left</button>
  3.     <button class="btn-primary active">Middle</button>
  4.     <button class="btn-primary">Right</button>
  5. </w-checkbox-button-group>
  6.  

Programatic

Havok uses AMD (Async Module Definition) throughout. To load and use a module, use require. Eg:

  1. <script type="text/javascript">
  2.     require([
  3.         'havok/widget/Modal', /*load the Modal module*/
  4.         'dojo/domReady!'      /*wait until the dom is ready before executing the function*/
  5.     ], function(Modal){
  6.         var modal = new Modal;                            /*create a new modal widget*/
  7.         modal.header.innerHTML = 'Hello';                 /*give the modal a title*/
  8.         modal.containerNode.innerHTML = 'My first modal'; /*give the modal some content*/
  9.         modal.startup();                                  /*startup the widget*/
  10.         modal.show();                                     /*show the modal*/
  11.     })
  12. </script>
