Using Forms

Form widgets

Havok provides a wide range of enhanced form elements for the input and validation of data. They can be used stand alone. However, use inside a form provides powerful data binding functions.

Data binding

Any havok form widges with a name attribute and wrapped in a f-form will automatically data bind. Eg:

please enter you email
please select a date

state:

        
value:
        
    
  1. <f-form>
  2.     <f-textbox name="name" label="Name" validator="Alpha" value="asdf"></f-textbox>
  3.     <f-email name="email" label="Email" help-messages="please enter you email" value="example@example.com"></f-email>
  4.     <f-date name="date" label="Date" help-messages="please select a date"></f-date>
  5. </f-form>


The value of all the contained widgets can be set and retrieved through the form's value property. Use:


        
  • myform.set('value', valueObj) to set a form's value.
    •     
  • myform.get('value') to get a form's value.
    •     
  • myform.watch('value', function) to watch a form's value.


Form state


The state reflects the state of the contained widgets. If the state is an empty string, the state is good. If the state is something else, the form is not valid. Use:


        
  • myform.set('state', valueObj) to set a form's state.
    •     
  • myform.get('state') to get a form's state.
    •     
  • myform.watch('state', function) to watch a form's state.

Message position


There are three possible values for message-position:


        
  • auto: if the message is one line, display inline. If it is multiline, display block
    •     
  • inline: always display message inline. If the message is more than one line, only the first will be shown.
    •     
  • block: always display message as block, even when there is only one line.
    
        
Please enter your full name
    


  1. <f-textbox label="name" message-position="block" help-messages="Please enter your full name"></f-textbox>


Required star
A required star will automatically be added to required form inputs. However, if you want to force a required star to show, use the required-star attribute.
    
        
    


  1. <f-textbox label="name" required-star></f-textbox>


Post Activity


All form widgets have a postActivity property which is set to true after user interaction.

Specifying the Validator


Havok uses the mystique validator library. And, havok/validator/factory to create validator instances. The validator property will be passed to the validator factory to construct a validator.


Create a validator with constructor options:
    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox validator='{"base": "Length", "params": {"min": 5, "max": 10}}' value="123"></f-textbox>



Validator chains
    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox validator='["Alpha", {"base": "Length", "params": {"min": 5, "max": 10}}]' value="123"></f-textbox>


Required and NotRequired Validators


If an input has a required attribute, a Required validator will automatically be added. If not, a NotRequired validator will automatically be added.


    
    
        
    
    
        
state:
        
        
value:
        
    
  1. <f-textbox required></f-textbox>


validationStyle
By default, validation errors are only highlighted after user interaction with the input. These styles be customised using the validationStyle property. It holds four lists of css class names that will be applied the four different validation situations.


    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox
  2.     validator="Alpha"
  3.     validation-style='{
  4.         "preActivity": {
  5.             "valid": ["info"],
  6.             "invalid": ["warning"]
  7.         },
  8.         "postActivity": {
  9.             "valid": ["info"],
  10.             "invalid": ["error"]
  11.         }
  12.     }'
  13.     value = "asdf123"
  14. ></f-textbox>


post-activity
To force the postActivity style to be applied, set the post-activity attribute.


    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox validator="Alpha" post-activity value="asdf123"></f-textbox>


delay
To stop validation occuring on every single keystroke, there is a validation delay. It is set to 350ms by default. The example below has very slow validation with delay set to 2000ms:


    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox validator="Alpha" delay=2000 value="asdf123"></f-textbox>


suppress-validation
The suppress-validation attribute will stop all validation. It will prevent updates to the state of the textbox.


    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox validator="Alpha" suppress-validation value="asdf123"></f-textbox>


suppress-validation-messages
The suppress-validation-messages attribute will allow validation to continue, but will not show validation messages. Message suppression can be set for pre and post activity. Messages are suppressed pre activity by default.


    
    
        
    
    
        
state:
        
        
value:
        
    


  1. <f-textbox
  2.     validator="Alpha"
  3.     suppress-validation-messages='{"preActivity": false, "postActivity": true}'
  4.     value="asdf123"
  5. ></f-textbox>
