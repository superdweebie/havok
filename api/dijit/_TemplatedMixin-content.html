          dijit/_TemplatedMixin

          Inherits from dijit/_AttachMixin

          Mixin for widgets that are instantiated from a template

          Usage

          _TemplatedMixin is a mixin. It is not intended to be used alone.

          To extend a class with _TemplatedMixin use declare. Eg:

          1. require(['dojo/_base/declare', 'my/parent/class', 'dijit/_TemplatedMixin'], function(declare, MyParent, _TemplatedMixin){
          2.     var MyChildClass = declare([MyParent, _TemplatedMixin], function(){
          3.         //other child extensions here
          4.     });
          5. Â 
          6.     var child = new MyChildClass; //create an instance of MyChildClass
          7. });

          _rendered

          _rendered - private

          Defined by dijit/_TemplatedMixin

          Types: Boolean

          Not normally use, but this flag can be set by the app if the server has already rendered the template, i.e. already inlining the template for the widget into the main page. Reduces _TemplatedMixin to just function like _AttachMixin.

          _attachTemplateNodes

          _attachTemplateNodes(rootNode) - private

          Defined by dijit/_AttachMixin

          Iterate through the dom nodes and attach functions and nodes accordingly.

          Map widget properties and functions to the handlers specified in the dom node and it&#39;s descendants. This function iterates over all nodes and looks for these properties:

          • dojoAttachPoint/data-dojo-attach-point
          • dojoAttachEvent/data-dojo-attach-event
          Parameter Type Usage Summary
          rootNode DomNode required

          The node to search for properties. All descendants will be searched.
