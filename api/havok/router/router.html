                              havok/router/router

                              Manage page changes using html5 pushState api.

                              The router lets you manage the browser location for simple dynamic page loading, without having to use # hacks. See router docs for usage examples.

                              Properties

                              All properties are filtered from view.

                              active

                              active

                              Defined by havok/router/router

                              Types: Object

                              Holds information about the currently active route.

                              An object with the following properties:

                              • ignore (Boolean):

                                If true, no matching route was found

                              • path (String)
                              • controller (String)
                              • entry (String)
                              • exit (String)
                              • args (String[]):

                                Any arguments passed to the controller entry method

                              baseUrlDefault

                              baseUrlDefault

                              Defined by havok/router/router

                              Types: String

                              If the baseUrl of the router is not explicitly given, this property defines how the baseUrl should be set.

                              If set to page, the baseUrl is set using windown.location.pathname when the router is started.

                              If set to site, the baseUrl is set using window.location.hostname when the router is started.

                              di

                              di

                              Defined by havok/router/router

                              Types: Object

                              Must be an instance of havok\di\Di. This is where controller instances will be pulled from.

                              routes

                              routes

                              Defined by havok/router/router

                              Types: Object

                              Array of Route objects that defines the mapping between urls and controller instances

                              An object with the following properties:

                              • regex (String):

                                Used to test if the requested path matches with this route.

                              • controller (String):

                                The name of a controller instance that can be retrieved through the di instance.

                              • defaultMethod (String|Object):

                                If a String, defines the name of the entry method to call on the controller if there is no method match. If an Object, it must have the form {entry: String, exit: String} and can define both the entry and exit methods to call on the controller if there is no method match.

                              • methods (Object):

                                An array of path to method name pairs.

                              Methods

                              All methods are filtered from view.

                              go

                              go(path, surpressPushState)

                              Defined by havok/router/router

                              change page location to the given path

                              Parameter Type Usage Summary
                              path String | int required

                              path may be a string, which will be routed, or a number which move to a relative point in history
                              surpressPushState Boolean optional

                              if true, the pushState api will not be used. ie, the address bar will not change

                              Returns: boolean

                              resolve

                              resolve(path)

                              Defined by havok/router/router

                              Attempt to resolve the provided path to a configured Route

                              Parameter Type Usage Summary
                              path String required

                              Returns: __RouteMatch | object

                              startup

                              startup()

                              Defined by havok/router/router

                              Start the router listening to click events.
