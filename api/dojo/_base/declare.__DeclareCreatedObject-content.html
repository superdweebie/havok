                    dojo/_base/declare.__DeclareCreatedObject

                    dojo/_base/declare() returns a constructor C. new C() returns an Object with the following methods, in addition to the methods and properties specified via the arguments passed to declare().

                    Methods

                    All methods are filtered from view.

                    createSubclass

                    createSubclass(mixins, props)

                    Defined by dojo/_base/declare

                    Create a subclass of the declared class from a list of base classes.

                    Create a constructor using a compact notation for inheritance and prototype extension.

                    Mixin ancestors provide a type of multiple inheritance. Prototypes of mixin ancestors are copied to the new class: changes to mixin prototypes will not affect classes to which they have been mixed in.

                    Parameter Type Usage Summary
                    mixins Function[] required

                    Specifies a list of bases (the left-most one is the most deepest base).
                    props Object optional

                    An optional object whose properties are copied to the created prototype.

                    Returns: dojo/_base/declare.__DeclareCreatedObject

                    Examples
                    1. var A = declare(null, {
                    2.     m1: function(){},
                    3.     s1: "bar"
                    4. });
                    5. var B = declare(null, {
                    6.     m2: function(){},
                    7.     s2: "foo"
                    8. });
                    9. var C = declare(null, {
                    10. });
                    11. var D1 = A.createSubclass([B, C], {
                    12.     m1: function(){},
                    13.     d1: 42
                    14. });
                    15. var d1 = new D1();
                    16.  
                    17. // this is equivalent to:
                    18. var D2 = declare([A, B, C], {
                    19.     m1: function(){},
                    20.     d1: 42
                    21. });
                    22. var d2 = new D2();
                    extend

                    extend(source)

                    Defined by dojo/_base/declare

                    Adds all properties and methods of source to constructor&#39;s prototype, making them available to all instances created with constructor. This method is specific to constructors created with declare().

                    Adds source properties to the constructor&#39;s prototype. It can override existing properties.

                    This method is similar to dojo.extend function, but it is specific to constructors produced by declare(). It is implemented using dojo.safeMixin, and it skips a constructor property, and properly decorates copied functions.

                    Parameter Type Usage Summary
                    source Object required

                    Source object which properties are going to be copied to the constructor&#39;s prototype.
                    Examples
                    1. var A = declare(null, {
                    2.     m1: function(){},
                    3.     s1: "Popokatepetl"
                    4. });
                    5. A.extend({
                    6.     m1: function(){},
                    7.     m2: function(){},
                    8.     f1: true,
                    9.     d1: 42
                    10. });
                    getInherited

                    getInherited(name, args)

                    Defined by dojo/_base/declare

                    Returns a super method.

                    This method is a convenience method for "this.inherited()". It uses the same algorithm but instead of executing a super method, it returns it, or "undefined" if not found.

                    Parameter Type Usage Summary
                    name String optional

                    The optional method name. Should be the same as the caller&#39;s name. Usually "name" is specified in complex dynamic cases, when the calling method was dynamically added, undecorated by declare(), and it cannot be determined.
                    args Arguments required

                    The caller supply this argument, which should be the original "arguments".

                    Returns: any | object

                    Examples
                    1. var B = declare(A, {
                    2.     method: function(a, b){
                    3.         var super = this.getInherited(arguments);
                    4.         // ...
                    5.         if(!super){
                    6.             console.log("there is no super method");
                    7.             return 0;
                    8.         }
                    9.         return super.apply(this, arguments);
                    10.     }
                    11. });
                    inherited

                    inherited(name, args, newArgs)

                    Defined by dojo/_base/declare

                    Calls a super method.

                    This method is used inside method of classes produced with declare() to call a super method (next in the chain). It is used for manually controlled chaining. Consider using the regular chaining, because it is faster. Use "this.inherited()" only in complex cases.

                    This method cannot me called from automatically chained constructors including the case of a special (legacy) constructor chaining. It cannot be called from chained methods.

                    If "this.inherited()" cannot find the next-in-chain method, it does nothing and returns "undefined". The last method in chain can be a default method implemented in Object, which will be called last.

                    If "name" is specified, it is assumed that the method that received "args" is the parent method for this call. It is looked up in the chain list and if it is found the next-in-chain method is called. If it is not found, the first-in-chain method is called.

                    If "name" is not specified, it will be derived from the calling method (using a methoid property "nom").

                    Parameter Type Usage Summary
                    name String optional

                    The optional method name. Should be the same as the caller&#39;s name. Usually "name" is specified in complex dynamic cases, when the calling method was dynamically added, undecorated by declare(), and it cannot be determined.
                    args Arguments required

                    The caller supply this argument, which should be the original "arguments".
                    newArgs Object optional

                    If "true", the found function will be returned without executing it. If Array, it will be used to call a super method. Otherwise "args" will be used.

                    Returns: any | object

                    Examples
                    1. var B = declare(A, {
                    2.     method1: function(a, b, c){
                    3.         this.inherited(arguments);
                    4.     },
                    5.     method2: function(a, b){
                    6.         return this.inherited(arguments, [a + b]);
                    7.     }
                    8. });
                    9. // next method is not in the chain list because it is added
                    10. // manually after the class was created.
                    11. B.prototype.method3 = function(){
                    12.     console.log("This is a dynamically-added method.");
                    13.     this.inherited("method3", arguments);
                    14. };
                    1. var B = declare(A, {
                    2.     method: function(a, b){
                    3.         var super = this.inherited(arguments, true);
                    4.         // ...
                    5.         if(!super){
                    6.             console.log("there is no super method");
                    7.             return 0;
                    8.         }
                    9.         return super.apply(this, arguments);
                    10.     }
                    11. });
                    isInstanceOf

                    isInstanceOf(cls)

                    Defined by dojo/_base/declare

                    Checks the inheritance chain to see if it is inherited from this class.

                    This method is used with instances of classes produced with declare() to determine of they support a certain interface or not. It models "instanceof" operator.

                    Parameter Type Usage Summary
                    cls Function required

                    Class constructor.

                    Returns: any | object

                    Examples
                    1. var A = declare(null, {
                    2.     // constructor, properties, and methods go here
                    3.     // ...
                    4. });
                    5. var B = declare(null, {
                    6.     // constructor, properties, and methods go here
                    7.     // ...
                    8. });
                    9. var C = declare([A, B], {
                    10.     // constructor, properties, and methods go here
                    11.     // ...
                    12. });
                    13. var D = declare(A, {
                    14.     // constructor, properties, and methods go here
                    15.     // ...
                    16. });
                    17.  
                    18. var a = new A(), b = new B(), c = new C(), d = new D();
                    19.  
                    20. console.log(a.isInstanceOf(A)); // true
                    21. console.log(b.isInstanceOf(A)); // false
                    22. console.log(c.isInstanceOf(A)); // true
                    23. console.log(d.isInstanceOf(A)); // true
                    24.  
                    25. console.log(a.isInstanceOf(B)); // false
                    26. console.log(b.isInstanceOf(B)); // true
                    27. console.log(c.isInstanceOf(B)); // true
                    28. console.log(d.isInstanceOf(B)); // false
                    29.  
                    30. console.log(a.isInstanceOf(C)); // false
                    31. console.log(b.isInstanceOf(C)); // false
                    32. console.log(c.isInstanceOf(C)); // true
                    33. console.log(d.isInstanceOf(C)); // false
                    34.  
                    35. console.log(a.isInstanceOf(D)); // false
                    36. console.log(b.isInstanceOf(D)); // false
                    37. console.log(c.isInstanceOf(D)); // false
                    38. console.log(d.isInstanceOf(D)); // true
